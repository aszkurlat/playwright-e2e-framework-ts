# Playwright E2E Framework (TypeScript)

This repository contains a complete **E2E automation framework** built with **Playwright + TypeScript**, demonstrating a real-world project structure based on Page Object Model (POM), fixtures, reusable test data, and suite tagging (smoke / regression).

The framework uses the **SauceDemo** application as the target system under test.

## ğŸ§© Page Object Model (POM)

Each page in the application is represented by a dedicated class:

### **LoginPage**
- login action  
- locators for username, password, error messages  
- navigation to main login page  

### **ProductsPage**
- add product to cart  
- sorting functionality  
- menu interactions  
- navigation to cart  

### **CartPage**
- validate cart page  
- remove item  
- proceed to checkout  
- check cart item count  

### **CheckoutPage**
- fill checkout form  
- validation for required fields  
- finishing the order  
- asserting order confirmation  

This clear separation makes tests small, readable, and easy to maintain.

## ğŸ” Authentication (storageState)

Initial authentication is performed once in:

global-setup.ts

It logs in with a valid user and saves session data to:

storageStates/standardUser.json

Tests that import the authenticated fixture:

```ts
import { test, expect } from "../../fixtures/authFixture";
```

start already logged in.

Tests that import:
```ts
import { test, expect } from "@playwright/test";
```
run without stored authentication and perform manual login inside the test.
